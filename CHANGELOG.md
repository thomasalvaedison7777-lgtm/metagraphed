# Changelog

Human-readable release notes for the hosted **metagraphed platform** — the
Cloudflare Worker API, MCP server, registry build, and agent-discovery surfaces.
From the next release onward this file is generated by release-please from
Conventional Commits (the `platform` component).

A few things this project versions differently:

- The hosted **API + artifacts** are continuously deployed; the REST/data
  contract is identified by the date-based `CONTRACT_VERSION` in
  `src/contracts.mjs` (e.g. `2026-06-06.1`). This changelog's `platform` SemVer
  is a separate, human-readable release marker for that continuous deploy — not a
  packaged release.
- The **MCP tool surface** has its own SemVer, `MCP_SERVER_VERSION`.
- The **published client SDKs** are versioned independently, with their own
  changelogs: npm [`packages/client/CHANGELOG.md`](packages/client/CHANGELOG.md)
  and PyPI [`python/CHANGELOG.md`](python/CHANGELOG.md).
- **Registry data enrichments** (new/updated subnets, providers, surfaces) are
  not listed here — they show up in the live `/api/v1/changelog` feed.

## 0.1.0

### Added

- Static agent tool specs for OpenAI + Anthropic at
  `/.well-known/agent-tools/{index,openai,anthropic}.json`, projected from the
  MCP tool list.
- Per-subnet / per-endpoint reliability score (0–100 + grade) over the durable
  uptime history, surfaced on `/api/v1/subnets/{netuid}/uptime` and MCP
  `get_subnet_health`.
- Global cross-subnet incident ledger at `GET /api/v1/incidents` (7d/30d window).
- A worked, schema-valid `example` on every OpenAPI operation, enforced in CI.
- Bidirectional MCP Registry backlink in the served server-card, `mcp.json`, and
  the live `initialize` result.
- The MCP server is listed in the canonical MCP Registry as
  `io.github.JSONbored/metagraphed`.
- API edge-caching for pure static-artifact GETs; captured live request/response
  fixtures (`get_fixture`); RPC pool-eligibility hysteresis + chain/genesis
  verification; a dedicated `MCP_SERVER_VERSION`.
- `good first issue` / `help wanted` labels on the enrichment queue;
  `CHANGELOG.md`, `FUNDING.yml`.

### Changed

- Deepened the previously-shallow OpenAPI response schemas to be fully typed
  down to the leaf fields.
- Provenance-tiered data storage (ADR 0006): retired the scheduled sync-subnets
  PR; machine data now refreshes via the publish into R2/D1.

### Fixed

- Bounded the `/api/v1/incidents` source query so it can't be used as a database
  load generator; adapter `extensions` values now type as open maps.
- Serve the fixtures index from R2 (was serving the empty committed copy).
